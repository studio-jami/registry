**Artifacts** are the outputs an agent produces — evidence packets, generated reports,
patches, screenshots, diagrams, and release outputs. Each artifact carries source bindings and
verification metadata so a result can be traced back to the work that produced it.

## Artifact Store

The default SDK composition uses an in-memory artifact store. Replace it with
`createHarness({ artifactStore })` when the deployment owns a durable store.

## Artifact Views

Artifact views are display-only references. They identify the artifact, source binding,
availability state, evidence ref, and resident presentation metadata without executing code.

## Evidence Link

Artifacts feed [evidence](/harness/concept-evidence), release readiness, and local workbench
views. Missing, stale, denied, and redacted artifact states are represented explicitly.
