Run the narrowest complete checks for the touched surface, then climb the ladder as you go.

## Ladder

- **Docs and plans** — read back changed Markdown, `pnpm docs:check`, `git diff --check`.
- **Generated docs/manual/system-map/changelog surfaces** — `pnpm docs:generate -- --check` plus `pnpm docs:check`.
- **Package metadata, scripts, generated surfaces, or CI** — docs checks plus the touched command or generator, then `pnpm verify`.
- **Contracts and schemas** — generation, drift check, schema validation, compatibility fixtures.
- **Runtime / policy / tools / memory / artifacts / observability** — lint, typecheck, unit tests, targeted integration tests, recovery or policy regression fixtures as applicable.
- **Current package gates** — `pnpm policy:test`, `pnpm runtime:test`, `pnpm tools:test`, `pnpm observability:test`.
- **Current CLI/SDK package gates** — `pnpm sdk:test`, `pnpm cli:test`, `pnpm examples:smoke`.
- **Current local regression eval smoke** — `pnpm eval:smoke`.
- **CLI / SDK / workbench** — command smoke, idempotency checks, clean temporary-project checks, browser smoke when UI exists.
- **Full gate** — `pnpm verify`, including the non-publishing release readiness and dry-run audits.

Every agent final response must report the exact checks run, results, unavailable commands, commit SHA, and push result.
