**Principled Edge** is the canon term for the Jami Harness posture: root-correct, zero-bloat, final-shape systems that stand on leading open tooling without surrendering the product vocabulary.

## What it means in practice

- The owned core comes first. Any concept that appears in user trust, agent autonomy, artifact promotion, public documentation, or governance claims belongs in the owned core before it belongs behind an adapter.
- Free, local tooling first. Prefer `pnpm`, TypeScript, Node scripts, `rg`, Mermaid, JSON Schema, OpenAPI, Vitest, Playwright, `git diff --check`, and generated docs from local manifests.
- Paid or hosted services are adapters, publishing targets, or fallback gates — never the only way to validate correctness.
- No stub completion. Do not fake progress with placeholders, hidden demo data, disabled checks, broad compatibility shims, or weakened validation.
- Evidence-backed claims only. Public claims must use the release-readiness claims matrix and current command evidence. Unsupported surfaces are named as unavailable instead of softened into marketing language.

## What it rules out

- Vendor lock at the trust surface.
- Hosted runtime claims without real command evidence.
- Hidden coupling between policy, tool execution, memory, and UI rendering.
- Drifting documentation that does not trace to source records.
