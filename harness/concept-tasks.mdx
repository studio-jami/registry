A **task** is a unit of work inside a [run](/harness/concept-runs). Tasks carry their own
context and memory and move through a defined lifecycle, so the work an agent does is broken
into auditable, recoverable steps rather than one opaque operation.

## Relationship to Runs

Run events may carry a `taskId`, and shared seam fixtures use task-scoped events for progress,
tool-running, checkpoint, retrying, recovered, and completed states. A task is therefore the
auditable work item inside the larger run record.

## Context and Memory

Task execution receives context from the configured context assembler and memory/search
ports. Context packs preserve inclusion reasons and deterministic hashes so the task can be
replayed and inspected.

## Tool Calls

Tasks invoke tools through [policy](/harness/concept-policy). Tool calls become traces,
audit records, artifacts, and evidence instead of hidden side effects.
