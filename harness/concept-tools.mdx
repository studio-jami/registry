**Tools** are how an agent acts on the world in Jami Harness. Every tool call is typed and
runs through a wrapper that enforces permissions, applies policy, records replay context, and
writes an audit record — so side effects are governed, not implicit.

## Supported Adapters

The current tool registry supports first-party function tools and trusted in-process MCP
fixture tools. Unsupported adapters fail closed: OpenAPI, shell, browser, code execution,
provider-as-tool, A2A, remote MCP stdio, and remote MCP Streamable HTTP.

## Tool Definitions

Function tools use `tool_` ids, declare risk and required scopes, and execute through the
policy-gated tool gateway. Tool results can create artifacts and traces, but the handler is
not allowed to bypass policy.

## Inspecting Tools

Use `jami tools --json` or `harness.inspect()` to see supported adapters, unsupported
adapters, and source-lock posture.
