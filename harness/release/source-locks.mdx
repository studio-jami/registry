When harness code depends on a drift-prone protocol, package, hosted service, registry format, guidance document, or release tool, it adds **source-lock evidence** for the exact surface used.

Each source-lock record captures:

- official URL
- version or spec id
- license / NOTICE / provenance status
- hash or tarball integrity (when applicable)
- command evidence
- unresolved risks
- next refresh trigger

Source-lock records are the gate that keeps Harness from drifting silently away from an upstream surface.
