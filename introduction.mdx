The Jami Studio Registry is the public distribution surface for [Studio UI](/studio-ui/overview) items: primitives, components, tokens, brand options, blocks, pages, apps, and full application suites. It is hosted at [registry.jami.studio](https://registry.jami.studio) and follows the [shadcn registry protocol](https://ui.shadcn.com/docs/registry).

The Registry is one of three pillars in the Jami Studio family. The other two are [Jami Harness](/harness/overview) (the contract-first agent runtime foundation) and [Studio UI](/studio-ui/overview) (the resident rendering layer whose items this registry distributes).

## What's in the Registry

The Jami Studio Registry hosts 45 items across several categories:

- **Primitive components** — Button, Panel, TextField, DataList, AgentPanel, DocsSourcePanel, MediaGrid
- **Suite bundles** — four pre-built application suites: Solo, Business Ops, Research Writing, and Mixed Media
- **Design tokens** — the Jami Factory theme with CSS variables, Tailwind integration, and TypeScript types
- **Brand options** — selectable accent/radius/spacing presets for different product personalities

All items follow the [shadcn registry protocol](https://ui.shadcn.com/docs/registry), so you can install them with the `shadcn` CLI directly.

## Registry Endpoint

The public registry is available at a stable URL with no authentication required:

```text
https://registry.jami.studio/registry.json
```

Use this URL as your registry source when running `shadcn add` commands.
