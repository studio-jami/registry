The Jami Studio registry validates every item against a JSON schema. This page will document
that contract — the item `type` values along the composition ladder
(`primitive → component → block → page → app → workspace`), the shared item fields, and the
workspace install-graph manifest shape.

## Planned reference

- The `type` enumeration and what each layer means.
- Core item fields (`name`, `title`, `files`, `registryDependencies`, and more).
- The `workspace` membership field and the workspace manifest (`*.workspace.json`).
