The Jami Studio component registry ships **seven token-driven UI primitives** that your application can install individually or as a group. Every component in the registry carries three first-class guarantees: a versioned prop schema that validates inputs at call-time, a declared set of design-token requirements that the CSS layer reads from `jami-ui.css`, and a complete accessibility specification wired directly into the rendered markup (roles, `aria-*` attributes, live regions, and focus rings). You do not need Tailwind, a CSS-in-JS runtime, or any shadcn package as a runtime dependency ‚ÄĒ the registry uses the shadcn-compatible install format purely to copy source into your project.

## Two rendering layers

Jami UI components are available through two complementary layers that you can mix freely:

  <Card title="Framework-neutral factory" icon="code">

They produce plain element descriptor objects ‚ÄĒ no React import required. Pass any `createElement`-compatible function from React, Preact, Solid, or your own renderer.

  <Card title="React / Radix wrappers" icon="react">
    Pre-built `forwardRef` components (`JamiButton`, `JamiPanel`, `JamiTextField`, `JamiDataList`, `JamiAgentPanel`, `JamiDocsSourcePanel`, `JamiMediaGrid`) live in `jami-radix-react-wrappers.mjs`. They require **React ‚Č• 19** and add Radix UI primitives for accessible label association and the `asChild` slot pattern.
  </Card>
</CardGroup>

## The component vocabulary

The registry's source of truth is `jami-vocabulary.mjs` (installed to `components/ui/jami-vocabulary.mjs`). It exports a frozen `componentVocabulary` array where every entry defines:

- **`propSchema`** ‚ÄĒ a JSON-Schema‚Äďstyle object with `properties`, `required`, and `additionalProperties: false`. The `validateComponentProps(name, props)` helper returns an array of human-readable issue strings before any rendering happens.
- **`tokens`** ‚ÄĒ the exact CSS custom-property namespaces the component reads from `jami-ui.css` (e.g. `semantic.light.accent`, `componentState.focusRing`).
- **`aria`** ‚ÄĒ the ARIA role, label source precedence, and any live-region or state attribute the component sets automatically.
- **`states`** ‚ÄĒ every interactive and content state the component is authored to handle (keyboard, focus-visible, disabled, loading, invalid, empty, error, and more).

<Note>
  All prop schemas use `additionalProperties: false`. Passing an undeclared prop returns a validation issue rather than silently forwarding unknown attributes to the DOM.
</Note>

## React ‚Č• 19 requirement

The pre-built Radix wrappers in `jami-radix-react-wrappers.mjs` rely on `React.forwardRef` and `React.isValidElement` from React 19. If you are using the framework-neutral `renderPrimitiveSpec` factory directly, there is no React dependency at all.

<Note>
  The `compatibility.react` field on every registry item is `">=19"`. Older React versions are not tested and are not supported for the Radix wrapper layer.
</Note>
