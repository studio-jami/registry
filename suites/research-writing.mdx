The Research Writing workspace is a focused knowledge-work shell for researchers, analysts, and writers who need to manage sources, build citation lists, draft outlines, and track writing-agent review status in a single coherent interface. It uses the `research-writing` workspace key and ships a `document-workbench` layout with four navigation items: Sources, Notes, Brief, and Outline. One install command resolves 14 registry items — theme tokens, primitives, the app shell, two pages, and four blocks — giving you a fully structured base for document-driven workflows.

## Install

Run the following command to pull the entire Research Writing workspace install graph into your project:

<CodeGroup>
```bash npm
npx shadcn add https://registry.jami.studio/items/research-writing-workspace.registry-item.json
```

```bash pnpm
pnpm dlx shadcn add https://registry.jami.studio/items/research-writing-workspace.registry-item.json
```

```bash yarn
yarn dlx shadcn add https://registry.jami.studio/items/research-writing-workspace.registry-item.json
```
</CodeGroup>

<Note>
  The install graph resolves all 14 dependencies automatically. You do not need to install individual items separately unless you want to use them outside the workspace.
</Note>

The `research-writing-workspace` registry item installs the following items in dependency order:

<Steps>
  <Step title="Design foundation">

  </Step>
  <Step title="Primitive components">

  <Step title="App shell">

  </Step>
  <Step title="Pages">

  </Step>
  <Step title="Blocks">
    `research-writing-sources-block`, `research-writing-citations-block`, `research-writing-outline-block`, `research-writing-agent-review-block` — the four discrete content blocks that compose each page.
  </Step>
</Steps>

## App Shell

The Research Writing app shell uses the `document-workbench` layout. This layout is optimised for reading and editing flows, with persistent navigation and a content area suited to long-form documents and reference panels displayed side by side.

| Property | Value |
|---|---|
| Shell ID | `app.research-writing.shell` |
| Layout | `document-workbench` |
| Workspace key | `research-writing` |
| Package name | `@jami-studio/research-writing-workspace` |

### Navigation

```
Sources  →  /research-writing/sources
Notes    →  /research-writing/notes
```

## Pages and Blocks

### Sources and citations — `/research-writing/sources`

Source display, citation lists, redaction-aware references, and long document titles. This page uses `docs-source-panel`, `data-list`, and `panel`.

<CardGroup cols={2}>
  <Card title="Sources" icon="book-open">
    `block.research-writing.sources` — powered by `docs-source-panel`. Exercises the `long-content` state: long source names, citation strings, and document titles wrap inside the writing shell without overflow.
  </Card>
  <Card title="Citations" icon="quote-right">
    `block.research-writing.citations` — powered by `data-list`. Exercises the `empty` state: when no citations are selected, the list displays an explicit empty table/list state rather than collapsing.
  </Card>
</CardGroup>

### Brief drafting — `/research-writing/brief`

Outline, notes, document state, and display-only writing-agent review. This page uses `docs-source-panel`, `text-field`, `agent-panel`, and `button`.

<CardGroup cols={2}>
  <Card title="Outline" icon="list-ol">
    `block.research-writing.outline` — powered by `text-field`. Exercises the `invalid` state so inline validation feedback is visible during development and testing.
  </Card>
  <Card title="Writing review" icon="robot">
    `block.research-writing.agent-review` — powered by `agent-panel`. Display-only writing-agent review; exercises the `error` state without executing any actions.
  </Card>
</CardGroup>

## State Fixtures

Each block ships with named state fixtures you can reference in tests and Storybook stories:

| Fixture | Description |
|---|---|
| `longContent` | Long source names, citation strings, and outline sections wrap inside the writing shell. |
| `empty` | No citations selected is displayed as an empty table/list state. |
| `error` | Review-agent errors are presented as inert status without executing actions. |
