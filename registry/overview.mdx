The Jami Studio Registry is a public, static shadcn-compatible registry served from [https://registry.jami.studio](https://registry.jami.studio). It contains 45 authored UI items — from low-level tokenized primitives to full application suite descriptors — each installable directly with the shadcn CLI. The registry follows the [shadcn registry protocol](https://ui.shadcn.com/docs/registry) so you can use standard tooling without any Jami-specific CLI.

## Registry endpoint

The top-level manifest listing all 45 items is available at:

```
https://registry.jami.studio/registry.json
```

The manifest conforms to the shadcn registry schema and is served statically from Cloudflare Pages. You do not need authentication to fetch it.

### Manifest shape

```json
{
  "$schema": "https://ui.shadcn.com/schema/registry.json",
  "name": "@jami-studio/registry",
  "homepage": "https://registry.jami.studio",
  "items": [
    {
      "name": "button",
      "type": "registry:ui",
      "title": "Button",
      "description": "Authored tokenized resident button/action-surface primitive.",
      "dependencies": ["@jami-studio/ui", "@radix-ui/react-slot@1.2.5"],
      "registryDependencies": [],
      "files": [ ... ]
    }
  ]
}
```

Each entry in `items` describes one installable unit. The fields are:

| Field | Description |
|---|---|
| `name` | Unique slug used in install commands and `registryDependencies` references |
| `type` | Registry type — see the type reference below |
| `title` | Human-readable display name |
| `description` | One-sentence summary of what the item provides |
| `dependencies` | npm packages the item requires at runtime |
| `registryDependencies` | Other registry item names that get pulled in automatically when you install this item |
| `files` | Array of file objects that are written to your project on install |

### File object shape

Each entry in `files` carries the complete source content inline, a source `path`, a `target` path (where the file lands in your project), and a `type` matching the parent item type:

```json
{
  "path": "packages/ui/src/vocabulary.mjs",
  "target": "components/ui/jami-vocabulary.mjs",
  "type": "registry:ui",
  "content": "...",
  "hash": "sha256:..."
}
```

The `hash` lets you verify file integrity after install.

## Item types

The registry uses the following shadcn registry type values:

| Type | Count | What it is |
|---|---|---|
| `registry:ui` | 3 | Low-level tokenized UI primitives (Button, Panel, Text Field) |
| `registry:component` | 4 | Composed resident components (Agent Panel, Data List, Docs Source Panel, Media Grid) |
| `registry:theme` | 4 | CSS variable token sets and selectable brand-option descriptors |
| `registry:block` | 18 | Single-component blocks inside a suite (agenda, task queue, review agent, etc.) |
| `registry:page` | 8 | Page-level compositions composed from blocks |
| `registry:app` | 4 | Full mounted React app implementation descriptors per suite |
| `registry:suite` | 4 | Top-level install-graph entry points that pull in a full application suite |

<Note>
  `registry:suite` items are the recommended entry point when you want a complete application shell. Installing a suite triggers the full install graph automatically — you do not need to install individual blocks and pages by hand. See the [Suites](/registry/suites) page for details.
</Note>
