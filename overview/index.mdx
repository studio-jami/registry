## How the family fits together

- **Harness** executes and governs agent work. It owns the action loop and emits typed,
  trustworthy references (run state, artifacts, action refs).
- **Studio** renders trusted state through resident, allowlisted UI. Registry items can
  *display* Harness-owned action and artifact references, but they never execute actions.
- **Workspaces** are the packaged products you install — each bundles one or more apps plus
  the pages, blocks, components, and a theme they need.

<Note>
  Harness and Studio UI have **no REST API**. Their developer interface is a TypeScript SDK,
  a CLI, and JSON-schema contracts. Each product's **Reference** section documents that real
  surface. (Intercal, a separate project, has its own REST API and its own docs site.)
</Note>
