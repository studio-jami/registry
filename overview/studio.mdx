**Jami Studio** is the UI foundation of the family. It publishes a public,
shadcn-compatible registry of UI items — from tokenized primitives up to complete,
installable **workspaces** — that you add to any React project with the standard `shadcn`
CLI.

## The composition ladder

Studio items compose from the smallest part up to the packaged product:

```
primitive → component → block → page → app → workspace
```

- **primitive** — an atomic UI part (Button, Panel, Text Field).
- **component** — a composed control (Agent Panel, Data List, Media Grid, Docs Source Panel).
- **block** — a page section.
- **page** — a full page composed from blocks.
- **app** — an application shell (routes + navigation); one or more per workspace.
- **workspace** — the top-level packaged product, customizable at install.

<Note>
  "Workspace" is the only product-level word. Studio items are display-only by design —
  action execution and capability truth stay in [Jami Harness](/overview/harness).
</Note>
