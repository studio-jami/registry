# Hosting And Docs Routing

Jami Registry uses one public hostname with two backing services:

- `registry.jami.studio/registry.json`, `/items/*`, `/suites/*`, and the generated workbench/suite preview routes are static files served by Cloudflare Pages.
- `registry.jami.studio/docs/*` is the user-facing docs route and should proxy to Mintlify.

Marketing stays at `jami.studio`.

## Current Status

- Static registry hosting is live at `https://registry.jami.studio/registry.json`.
- The npm packages are published at `0.1.0`.
- `https://registry.jami.studio/docs` is served by the Pages advanced-mode worker in `public/_worker.js`.
- `https://registry.jami.studio/` serves the generated Studio UI workbench preview.
- `https://registry.jami.studio/suites/solo/`, `/suites/business-ops/`, `/suites/mixed-media/`, and `/suites/research-writing/` serve generated suite previews.

## Cloudflare Subpath Proxy

Route `registry.jami.studio/docs*` through the Pages advanced-mode Worker in `public/_worker.js`.
The worker keeps Mintlify as the primary docs origin and falls back to local
generated preview docs for `/docs/workbench` and `/docs/suites` if Mintlify
returns 404.

Use the real Mintlify hostname from the project settings. The current origin is `jami.mintlify.dev`:

```js
const DOCS_ORIGIN = "jami.mintlify.dev";
const PUBLIC_HOST = "registry.jami.studio";

export default {
  async fetch(request, env) {
    const url = new URL(request.url);

    if (url.pathname === "/docs" || url.pathname.startsWith("/docs/")) {
      const upstream = new URL(url);
      upstream.hostname = DOCS_ORIGIN;

      const proxyRequest = new Request(upstream, request);
      proxyRequest.headers.set("Host", DOCS_ORIGIN);
      proxyRequest.headers.set("X-Forwarded-Host", PUBLIC_HOST);
      proxyRequest.headers.set("X-Forwarded-Proto", "https");

      return fetch(proxyRequest);
    }

    return env.ASSETS.fetch(request);
  },
};
```
